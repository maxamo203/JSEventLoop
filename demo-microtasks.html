¿El navegador repinta ENTRE microtasks?

El número de abajo lo vamos a actualizar muchas veces seguidas dentro de microtasks (Promises ya resueltas). Si el render ocurriera entre cada microtask, verías el número "contar" 1, 2, 3… animándose. Si el render espera a que se vacíe TODA la cola de microtasks, vas a ver que salta directo al valor final.

Experimento A — 50 microtasks que actualizan el DOM

Encadena 50 Promise.resolve().then(). Cada una escribe el número en pantalla y hace un poco de trabajo. Abrí la consola (F12) para ver el log con timestamps.

Experimento B — lo mismo pero con setTimeout (macrotasks)

Acá cada actualización es un setTimeout(…, 0). Como cada una es una macrotask distinta, el navegador SÍ puede repintar entre ellas. Comparalo con el A.
